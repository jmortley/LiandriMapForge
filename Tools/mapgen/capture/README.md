# Ground-truth capture

Geometry (BSP box brushes) is source-verified and generated by `../emit.py`. Two
things still need to be confirmed against a *real* editor export rather than
reconstructed blind, because each is easy to get subtly wrong and impossible to
test without the editor:

1. **Actor transform serialization** — PlayerStart / pickup / light store their
   location+rotation on a *root component subobject* whose name differs per class.
   We need the real `Begin Object ... End Object` block each one emits.
2. **Material paths** — the real `/Game/RestrictedAssets/.../M_*` paths UT4 uses,
   so generated walls aren't default-checker.

## Do this once (about 60 seconds in UnrealEd)

1. New empty level.
2. **BSP panel → drag in a Box** (additive). Leave it at default.
3. Drag in a **UTPlayerStart**, a **UTPickupHealth**, and a **Point Light**.
4. Select a real wall/floor static mesh from any shipping map (optional) so we
   capture a live material path.
5. Select all of them → **Ctrl+C** → paste into `ground-truth.t3d` in this folder.

That single paste gives us the exact actor wrappers + material paths, which lock
the emitter's actor emission to 4.15 reality. Drop the file here and say the word.

## Why geometry didn't need this

`../emit.py`'s brush blocks are matched line-for-line against this fork's own
exporter/importer (`EditorExporters.cpp:295`, `EditorObject.cpp:293-320,641`),
and `BrushType`/`BrushComponent0` were read from `Brush.h`/`Brush.cpp`. The paste
test (`out/smoke_cube.t3d`) confirms it end-to-end; if it imports, geometry is done.
